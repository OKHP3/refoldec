# 🧬 ReFolDec — The Recursively Folding Codec

> **ReFolDec** — the **Re**cursively **Fol**ding Co**dec** — is a bidirectional process-capture and transformation framework for folding raw thought into structured artifacts, then unfolding mature artifacts back into their source primitives, patterns, and reusable instructions.

---

## What it is

ReFolDec is a **capture-first system** for turning thoughts, workflows, narratives, and repeated actions into structured documentation, linked diagrams, reusable agent skills, and versioned canon.

It is not a single tool. It is a **framework of planes, layers, and mechanics** that governs how living process objects are captured, matured, inverted, visualized, and made executable — by humans, machines, or both.

---

## The two directions

| Direction | Name | What happens |
|---|---|---|
| ↓ Fold | **xME** (Maturation) | Raw thought → structured artifact → durable canon |
| ↑ Unfold | **xIE** (Inversion) | Mature artifact → source primitives → reusable instructions |

These two mechanics together form **xMIE** — the underlying operating physics of ReFolDec.

---

## The architecture

```
Ephemeral thought / human action / conversation
  ↓
Capture plane: Notion, transcript, Markdown, meeting notes
  ↓
Structuring layer: schema, page properties, frontmatter, ledgers
  ↓
Maturation layer: xME — expansion, synthesis, editorial passes
  ↓
Inversion layer: xIE — decomposition, primitive extraction, reusable pieces
  ↓
Executable layer: Agent Skills, prompts, runbooks, SOPs
  ↓
Visual layer: Mermaid, BPMN-for-Mermaid, linked diagrams
  ↓
Version/publish layer: GitHub, OverKill Hill, static sites
```

See [`ARCHITECTURE.md`](./ARCHITECTURE.md) for the full design.
